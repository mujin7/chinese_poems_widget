# 个性化中文诗词推荐系统

这是一个从零开始构思的个人项目，目标不是做一个普通诗词展示页，而是做一个以 **推荐系统、用户行为闭环、AI 内容理解、多端强触达** 为核心的中文诗词推荐产品。

## 项目目标

项目希望验证这样一个产品命题：

> 用户完成初始问卷后，系统根据用户的阅读、点击、收藏、喜欢、不喜欢、跳过等行为持续学习偏好，并通过 PWA、Web Push、浏览器新标签页插件等入口主动触达用户，推荐兼具熟悉感与多样性的中国古代诗词。

## 为什么做这个项目

这个项目同时服务两个目标：

1. **产品目标**：探索一个能够在日常生活中持续触达用户的个性化诗词推荐产品。
2. **求职展示目标**：作为推荐算法、AI 产品、AI 全栈开发、Agent 开发方向的作品，展示前后端连接、推荐系统搭建、AI/Agent 工作流和产品判断能力。

## 核心产品原则

- **自动化推荐**：系统必须根据用户行为持续更新推荐结果。
- **强触达**：产品不能依赖用户每天主动打开网站，需要通过通知、新标签页、组件或插件等形式进入用户日常路径。
- **熟悉与多样性平衡**：推荐既不能全是教材名篇，也不能全是陌生冷门作品。
- **可解释推荐**：用户应能看到“为什么推荐这首诗”。

## 计划中的产品形态

- **PWA Web App**：主产品入口，承载问卷、推荐流、详情页、收藏、历史和偏好设置。
- **Web Push**：每日推荐通知，用于移动端和桌面端强触达。
- **浏览器新标签页插件**：桌面端高频触达入口，每次打开新标签页展示个性化推荐。
- **壁纸生成**：作为推荐结果的衍生功能，而不是主产品形态。

## 计划中的技术方向

- 前端：Next.js / React / TypeScript
- 后端：FastAPI / Python
- 数据库：PostgreSQL / pgvector
- 推荐：标签召回、语义召回、排序、重排、多样性控制
- AI：LLM 标注、embedding、推荐解释、用户偏好总结
- Agent：内容标注 Agent、推荐解释 Agent、评估 Agent
