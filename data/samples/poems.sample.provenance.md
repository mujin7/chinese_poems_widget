# `poems.sample.jsonl` provenance

## Short answer

`data/samples/poems.sample.jsonl` is a **manual schema fixture**, not an imported corpus dump.

The 20 poem records were manually entered by the project agent from widely known public-domain Chinese classical poems to exercise the canonical `poem.v0` schema. They were created to make Step 2 testable before the upstream normalization script exists.

## What this sample is for

This file is intended to validate data shape and downstream compatibility:

- stable `poem_id` format;
- `content.paragraphs` and `content.lines` shape;
- `features.raw_mapped`, `features.rule_derived`, `features.weak_signals`, `features.semantic`, and `features.behavior_derived` groups;
- `source`, `review`, and `ai_annotations` traceability fields;
- validator behavior in `scripts/data_import/validate_poems.py`.

## What this sample is not

This file is **not** the final corpus source and should not be described as imported from `chinese-poetry/chinese-poetry`.

It is also not enough for production-quality text accuracy, because the records were not generated by an automated upstream import pipeline and have not yet been line-by-line compared against a source edition.

## Why this is acceptable for Step 2

Step 2 is about defining the canonical schema contract. A small manual fixture is useful because it lets us validate structure before building the normalizer.

However, this fixture must not become the product corpus. Step 3 should produce `data/processed/poems.seed.v0.jsonl` from upstream sources through a repeatable script.

## Required next step

Before using these poems as product content, each record must be one of:

1. replaced by a record generated from the upstream normalization script, or
2. explicitly verified against a documented source and marked with `review.text_status = "source_verified"`.

Until then, every record keeps:

```json
{
  "source_id": "manual_schema_fixture",
  "data_entry_method": "manual_entry_by_project_agent",
  "source_verification_status": "not_source_verified"
}
```
